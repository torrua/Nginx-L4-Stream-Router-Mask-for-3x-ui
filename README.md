Nginx L4 Stream Router & Mask for 3X-UI / Xray (v6.9.4)
Автоматизированный скрипт для развертывания отказоустойчивой, высокопроизводительной и безопасной маскировки VPN-сервера на базе Nginx (Stream L4 Router) и панели 3X-UI.

Скрипт полностью совместим с архитектурами x86_64 и ARM (включая Armbian и Ubuntu 24.04 Noble), оптимизирован как для производительных Ryzen VPS, так и для одноплатных компьютеров.

🌟 Основные фичи и архитектура
Единый порт 443/TCP для всего: На внешнем порту 443 висит умный Nginx Stream. Он анализирует SNI-заголовки запросов (технология ssl_preread) без расшифровки трафика:

Запросы к основному домену отправляются на локальный веб-сервер Nginx для отображения сайта-заглушки (декоя).

Запросы к секретному пути панели проксируются на внутренний веб-интерфейс 3X-UI.

Запросы к альтернативным доменам прозрачно перенаправляются на соответствующие инбаунды VLESS Reality.

Выделенный внутренний сервер подписок (v6.9.4+): Полностью решена проблема конфликтов портов и цикличных редиректов подписок. Встроенный сервер подписок панели изолирован на внутреннем порту (по умолчанию 55443), а Nginx берет на себя внешнее SSL-шифрование на порту 443.

Прямой биндинг Hysteria 2 (443/UDP): Трафик Hysteria 2 идет в обход Nginx напрямую в Xray, что гарантирует максимальную скорость и минимальный пинг для UDP.

Автоматизация SSL: Интеграция с официальным Certbot (через Snap) для автоматического выпуска и продления Let's Encrypt сертификатов с поддержкой SAN (Multi-Domain).

Кастомный Decoy: Встроенный интерактивный шаблон маскировки, имитирующий интерфейс авторизации приватного облака CosmosCloud.

🚀 Быстрый запуск
Склонируйте репозиторий или скачайте скрипт:

Bash
wget https://raw.githubusercontent.com/Itman75/OneScript_for_Install_3x-ui-Nginx-Certbot/refs/heads/main/setup_mask.sh
chmod +x setup_mask.sh
Запустите скрипт от имени root:

Bash
sudo ./setup_mask.sh
Следуйте инструкциям интерактивного мастера на экране.

🛠 Пошаговая настройка после работы скрипта
После успешного выполнения скрипта вам необходимо вручную настроить инбаунды и систему подписок в веб-интерфейсе 3X-UI.

1. Настройка инбаундов VLESS REALITY
Для каждого указанного при установке порта Reality создайте отдельный инбаунд в панели:

Порт: Укажите один из ваших портов Reality (например, 54320).

Listen IP: Строго впишите 127.0.0.1 (это скроет порты Reality от сканирования из внешнего интернета).

Reality -> Dest / Server Names: Укажите домен-декой стороннего ресурса и альтернативный домен, привязанный к этому порту.

Примечание для клиентов: В конфигурации клиента порт подключения всегда должен быть 443, а SNI — ваш альтернативный домен.

2. Настройка инбаунда Hysteria 2 (UDP)
Порт: 443

Протокол: udp

Listen IP: 0.0.0.0

Пути к сертификатам (Cert / Key): * Публичный ключ: /etc/letsencrypt/live/ВАШ_ГЛАВНЫЙ_ДОМЕН/fullchain.pem

Приватный ключ: /etc/letsencrypt/live/ВАШ_ГЛАВНЫЙ_ДОМЕН/privkey.pem

3. Настройка системы подписок (Важно!)
Чтобы ссылки на подписки генерировались корректно через порт 443 и не вызывали ошибок веб-сервера, примените следующие настройки в панели:

Перейдите в Настройки панели -> вкладка Настройки подписок.

Поставьте галочку Включить подписку (Enable Subscription).

В поле Порт подписки (Subscription port) впишите ваш выделенный внутренний порт: 55443 (или тот, что вы выбрали в Шаге 2 скрипта).

В поле URI-путь подписки (Subscription path) строго укажите: /postkey/.

В поле URI обратного прокси / Шаблон URL (Subscription URL template) пропишите адрес вашего домена без указания портов:

Plaintext
https://ВАШ_ГЛАВНЫЙ_ДОМЕН/postkey/%uuid%
(Или укажите базовый адрес обратного прокси как https://ВАШ_ГЛАВНЫЙ_ДОМЕН/postkey/)

Критически важно: Перейдите в раздел управления сертификатами для подписок (если он отображается отдельно) и убедитесь, что поля путей к SSL-сертификатам пусты. Внутренний сервер подписок должен работать по обычному HTTP.

Нажмите Сохранить настройки и затем нажмите Перезапустить панель.

🔒 Настройка Файрвола (UFW / iptables)
Для обеспечения максимального уровня стелс-режима закройте все служебные порты от внешнего мира.

Разрешить для внешних подключений (WAN):

80/TCP — для проверки Certbot.

443/TCP — основной вход для сайтов, панели, подписок и VLESS.

443/UDP — для работы Hysteria 2.

[Ваш кастомный порт SSH] — для управления сервером.

Заблокировать для внешних подключений:

Порт панели 3X-UI (например, 10443).

Внутренний порт подписок (55443).

Все локальные порты Reality (54320, 54322 и т.д.).
